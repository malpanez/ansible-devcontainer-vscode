Fix two structural gaps in ci.yml that prevent CI-02 and CI-03 from being satisfied:

1. The `molecule` job `if` condition gates on `main` only — develop commits never exercise molecule tests.
2. The `unit-tests` job uses a hardcoded Python version — compat regressions across Python 3.11/3.12 go undetected.

Purpose: Ensure every develop commit exercises the full test surface (including molecule) and that Python version regressions are caught early.
Output: Updated `.github/workflows/ci.yml` with both structural fixes applied.

<task type="auto">
  <name>Task 1: Fix molecule job trigger to include develop branch</name>
  <files>.github/workflows/ci.yml</files>
  <action>
In the `molecule` job, the current `if` condition (lines 539-541) is:

```yaml
if: |
  needs.changes.outputs.ansible == 'true' &&
  (github.ref == 'refs/heads/main' || github.event_name == 'workflow_dispatch')
```

Change it to also fire on `develop`:

```yaml
if: |
  needs.changes.outputs.ansible == 'true' &&
  (github.ref == 'refs/heads/main' || github.ref == 'refs/heads/develop' || github.event_name == 'workflow_dispatch')
```

Also apply the same fix to the `scenario-tests` job (lines 594-595) which has an identical main-only gate. Its condition is:

```yaml
if: |
  github.ref == 'refs/heads/main' || github.event_name == 'workflow_dispatch'
```

Change to:

```yaml
if: |
  github.ref == 'refs/heads/main' || github.ref == 'refs/heads/develop' || github.event_name == 'workflow_dispatch'
```

No other changes to these jobs. Preserve all indentation (2-space YAML). Do not alter the `needs`, `timeout-minutes`, `strategy`, or `steps` blocks.
  </action>
  <verify>
    <automated>grep -n "refs/heads/develop" /workspace/.github/workflows/ci.yml</automated>
  </verify>
  <done>Both `molecule` and `scenario-tests` jobs have `refs/heads/develop` in their `if` conditions. `grep` returns at least 2 matching lines.</done>
</task>

<task type="auto">
  <name>Task 2: Add Python version matrix to unit-tests job</name>
  <files>.github/workflows/ci.yml</files>
  <action>
In the `unit-tests` job (starting around line 442), convert the fixed Python setup to a matrix. The job currently uses `python-version: ${{ env.PYTHON_VERSION }}` with no matrix.

Add a `strategy` block directly after the `timeout-minutes` line and before `permissions`:

```yaml
    strategy:
      fail-fast: false
      matrix:
        python-version: ["3.11", "3.12"]
```

Update the job `name` to include the matrix version:
```yaml
  name: Python Unit Tests (Python ${{ matrix.python-version }})
```

Update the `Set up Python` step to use the matrix variable instead of the env var:
```yaml
      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@83679a892e2d95755f2dac6acb0bfd1e9ac5d548  # v6.1.0
        with:
          python-version: ${{ matrix.python-version }}
```

All other steps in the job remain unchanged. The `env.PYTHON_VERSION` global env var is still used by other jobs — do not remove it.

Also update `ci-success` job's `needs` list: `unit-tests` is already listed there. Since the job now fans out to a matrix, GitHub treats each matrix job as a separate check — the `ci-success` aggregation will still work correctly because the `needs` reference to `unit-tests` waits for ALL matrix legs. No change needed to `ci-success`.
  </action>
  <verify>
    <automated>grep -A5 "Python Unit Tests" /workspace/.github/workflows/ci.yml | grep "matrix.python-version"</automated>
  </verify>
  <done>The `unit-tests` job has a `strategy.matrix.python-version` with `["3.11", "3.12"]` and the `setup-python` step references `matrix.python-version`. `grep` confirms the matrix variable is present.</done>
</task>
